# Director - AI 影视创作工作流

> 以好莱坞导演思维，驾驭 AI 视频生成模型，打造专业级影视内容

## 项目简介

本项目是一套完整的 AI 辅助视频创作工作流系统，整合了即梦 Seedream/Seedance 等 AI 生成模型，结合专业电影制作理论，帮助创作者高效产出高质量影视内容。

## 阶段一：构思与策划

### 1.1 明确创作目标

| 维度 | 问题清单 |
|------|----------|
| 主题 | 核心表达什么？传递什么情感/价值观？ |
| 受众 | 目标观众是谁？他们的偏好和期待？ |
| 时长 | 短视频(15-60s)、中视频(1-5min)、长片(5min+)？ |
| 风格 | 写实、动画、混合？什么视觉调性？ |
| 平台 | 抖音/B站/YouTube/电影？不同平台的技术要求？ |

### 1.2 故事框架搭建

根据视频时长选择合适的叙事结构：

#### 短视频（15-60秒）

**钩子-核心-收尾** — 最常用
```
钩子(1-3s)     核心(主体)      收尾(2-3s)
   │              │               │
 抓住注意力    展示核心内容    反转/升华/CTA
```

**对比反转** — 适合剧情类
```
前半段：建立预期（A状态）
   ↓
转折点：打破预期
   ↓
后半段：揭示真相（B状态）
```

**单一冲突** — 适合情绪类
```
冲突呈现 ──> 情绪释放
```

#### 中视频（1-5分钟）

**起承转合** — 经典四段式
```
起(15%)      承(35%)       转(35%)       合(15%)
  │            │             │             │
建立情境    推进发展      转折高潮      收尾升华
```

**问题-探索-答案** — 适合知识/解说类
```
抛出问题 ──> 层层探索 ──> 给出答案
```

**并列递进** — 适合展示/盘点类
```
要点1 ──> 要点2 ──> 要点3 ──> 总结升华
```

### 输出物
- `scripts/[项目名]/story_outline.md` - 故事大纲

## 阶段二：剧本创作

### 2.1 场景列表

```markdown
## 场景列表

| 场景号 | 场景名称 | 时间 | 地点 | 角色 | 简述 |
|--------|----------|------|------|------|------|
| 1 | 开场 | 白天 | 城市街道 | 主角 | 主角走在繁华街头 |
| 2 | 转折 | 黄昏 | 咖啡馆 | 主角、导师 | 遇见改变命运的人 |
```

### 2.2 剧本格式

```
场景1 - 城市街道 - 白天 - 外景

【环境】繁华的现代都市，车水马龙，霓虹闪烁

主角（男，25岁）穿过拥挤的人群，神情疲惫。

主角（画外音）
"每一天都是重复的，直到那个下午..."

镜头缓慢推进主角面部特写。
```

### 输出物
- `scripts/[项目名]/screenplay.md` - 完整剧本

## 阶段五：分镜设计

### 5.1 分镜脚本格式

```markdown
## 分镜脚本

### 场景1：开场

| 镜号 | 时长 | 景别 | 运镜 | 画面描述 | 声音 | 备注 |
|------|------|------|------|----------|------|------|
| 1-1 | 3s | 全景 | 固定 | 城市天际线，晨光初照 | 环境音 | 定场镜头 |
| 1-2 | 2s | 中景 | 推进 | 主角走在街道上 | 脚步声 | |
| 1-3 | 2s | 特写 | 固定 | 主角疲惫的眼神 | 心跳声 | 情绪铺垫 |
```

### 5.2 分镜草图（可选）

可使用 AI 快速生成分镜草图作为参考。

### 5.3 多镜头叙事提示词

```
这个场景以第三人称视角，讲述[场景概述]。

第1个镜头[0-3秒] [景别]，[运镜]，[画面描述]。
第2个镜头[4-6秒] [转场方式]，[景别]，[画面描述]。
第3个镜头[7-10秒] [转场方式]，[景别]，[画面描述]。
```

### 输出物

## 阶段六：AI 素材生成

### 6.1 工作流选择

```
┌────────────────────────────────────────────────────┐
│              选择生成方式                           │
├────────────────────────────────────────────────────┤
│                                                    │
│   文生视频 ─────> 从零开始，完全由提示词控制        │
│       │                                            │
│   图生视频 ─────> 已有关键帧，控制起始画面          │
│       │                                            │
│   参考生视频 ───> 角色一致性，使用角色参考图        │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 6.2 提示词公式

**文生视频（完整控制）**
```
[景别], [光源], [光线类型], [时间段], [色调], [构图],
[主体描述], [动作/状态], [场景环境],
[运镜描述], [风格化]
```

**图生视频（动态控制）**
```
[运动描述] + [运镜描述]
```

**角色扮演（参考生视频）**
```
@主角 [动作描述], [台词], [场景]
```

### 6.3 生成检查清单

- [ ] 提示词是否包含足够的视觉细节？
- [ ] 运镜与情绪是否匹配？
- [ ] 角色一致性是否保持？
- [ ] 生成参数是否正确（分辨率、时长、帧率）？
- [ ] 是否需要多次迭代优化？

### 6.4 批量生成策略

1. **先静后动**：先生成关键帧图片，确认画面后再生成视频
2. **分段生成**：长镜头拆分为多个短片段
3. **备选方案**：每个镜头生成2-3个版本备选

### 输出物
- `outputs/[项目名]/[场景号]/` - 生成的视频/图片素材

## 阶段七：后期制作

### 7.1 剪辑工作流

```
1. 素材整理 ──> 按场景/镜头分类
2. 粗剪 ──────> 按分镜顺序排列
3. 精剪 ──────> 调整节奏和转场
4. 调色 ──────> 统一视觉风格
5. 音频 ──────> 配音、音效、音乐
6. 输出 ──────> 根据平台要求导出
```

### 7.2 推荐工具

| 环节 | 推荐工具 |
|------|----------|
| 剪辑 | DaVinci Resolve / Premiere Pro / 剪映 |
| 调色 | DaVinci Resolve |
| 音频 | Audition / 剪映 |
| 字幕 | Arctime / 剪映 |
| AI配音 | 即梦声音生成 / 11Labs |

### 7.3 输出规格参考

| 平台 | 分辨率 | 帧率 | 格式 |
|------|--------|------|------|
| 抖音/快手 | 1080x1920 | 30fps | MP4 |
| B站 | 1920x1080 | 60fps | MP4 |
| YouTube | 3840x2160 | 60fps | MP4 |
| 电影 | 4096x2160 | 24fps | ProRes |

## 快速开始

### 1. 创建新项目

```bash
# 创建项目目录结构
mkdir -p scripts/my_project
mkdir -p characters
mkdir -p scenes
mkdir -p storyboards/my_project
mkdir -p outputs/my_project
```

### 2. 从故事大纲开始

创建 `scripts/my_project/story_outline.md`，写下你的故事核心。

### 3. 逐步完善

按照工作流各阶段，依次完成：
- 剧本 → 角色 → 场景 → 分镜 → 生成 → 成片

---

## 最佳实践

### 创意阶段
- 先构建完整故事框架，再细化分镜
- 使用参考图片建立视觉方向
- 保持角色和场景的设定一致性

### 生成阶段
- 提示词要具体、精确、可视化
- 善用美学控制词典中的专业术语
- 多次迭代，逐步逼近理想效果
- 保存所有版本，便于后期选择

### 后期阶段
- 建立统一的色彩 LUT
- 注意镜头间的节奏和呼吸感
- 声音设计同样重要
